Method Detail
drawSkin()method
public final function drawSkin():void

Implements an empty body for the drawSkin interface.

Since the Skin uses the view life cycle, its draw() method is used instead.

Commands the skin to draw its content.

The method is invoked after the skin has been added to the parent display object and every time a skin should be updated, e.g. if its size changed.

Updating a skin usually requires a full redraw. The drawSkin() method of an ISkin implementor should therefore remove all content before it redraws.

init()method 
override protected final function init():void

Breaks the inheritance chain for the init() method.

Properties for a Skin have to be determined within the draw method.

Initialises view styles and properties.

This method is called once right after the view has been added to the display list.

Here is the first place to access parent style properties. You use this method to perform operations connected with view styles or properties, e.g. caching style values into a local properties or evaluating property consistencies.

styleChanged()method 
override protected final function styleChanged(property:String, value:*):void

Calls invalidate() automatically after a style change notification has been received. Notifies the view about a change of a style value for that it defines a default style.

To generally get notifications from the style manager, you need to declare all particular styles via the StyleManager.setDefaultStyles() method. You do this at best in the constructor of the client.

If this method has been called, you should update your view to reflect the changes in its visualisation.

Parameters

property:String — The style name.
 
value:* — The style value.

update()method 
override protected final function update():void

If a style changes, the skins is redrawn completely. If only the size has changed, a subclass may implement its custom resize method such as a scale9 scaling. Updates, redraws and layouts sub views.

draw() and update() have the same job to set the view in the right state. While the draw() method gets called only one, the update() method is invoked every time a validation cycle has been passed.

If you have started a validation cycle by calling invalidateProperty(property), isInvalid(property) will here return true.

After this method has finished, all invalidate properties are nullified.

updateSize()method 
protected function updateSize():void

If not overridden, this method redraws all content.

Override the method to work with scale9grid scaling rather than redrawing all content completely.
